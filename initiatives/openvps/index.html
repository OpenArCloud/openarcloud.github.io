The open spatial web starts here.

OpenVPS is the open-source Visual Positioning Service reference implementation from Open AR Cloud. It follows the OGC GeoPose standard and the OSCP GeoPoseProtocol so teams can self-host camera-based 6DoF localization without vendor lock-in or proprietary APIs.

A project of Open AR Cloud Europe, built by Nokia Bell Labs, and funded through NGI Atlantic and NGI Search. It is a batteries-included research tool and deployment foundation, not a managed cloud service.

What it is

An open, standards-compliant foundation for spatial positioning.

OpenVPS gives developers, researchers, and partner institutions a deployable VPS stack they can run on their own infrastructure. The site is intentionally direct: the GitHub link is prominent, the architecture is documented, and the provenance is explicit.

The project exists because visual positioning sits underneath the spatial web, and that layer should not belong exclusively to any single platform gatekeeper. OpenVPS makes the open alternative concrete.

No vendor lock-in No proprietary APIs No data leave your server Production-grade for research and R&D
What you get

Three capabilities that matter to teams building real systems.

Open standard

OpenVPS is compliant with OGC GeoPose and the OSCP GeoPoseProtocol. Any compliant client can talk to the same endpoint, so switching backends does not require rewriting the app.

Self-hosted

Your maps, your infrastructure. Deploy on-premises, at the edge, or on a private cloud using a Docker-based stack designed for teams who need control and repeatability.

Full pipeline

Capture, feature-map, align, and localize in one system. The OpenVPS stack includes the tools needed to go from raw imagery to a live GeoPose response.

Copy-ready demo client
python demo_client.py \
  --url https://YOUR_VPS_URL/localize/geopose \
  --image ./data/query.jpg \
  --camera ./data/camera_params.json \
  --geolocation ./data/geolocation_params.json
How it works

The pipeline is explicit, inspectable, and repeatable.

Step 01

Capture

Use the StrayScanner or Android Kapture app to collect RGB, depth, GPS, and sensor data. Upload the package into the builder.

→
Step 02

MapBuilder

Runs the HLOC pipeline to generate a feature map. The output becomes a spatial representation that can be aligned and served.

→
Step 03

MapAligner

Align the local point cloud to OpenStreetMap footprints and produce the geo-alignment transform that anchors the map to WGS84.

→
Step 04

MapLocalizer

The live server receives the image and coarse location, runs hierarchical localization, and returns a GeoPose response for any OSCP client.

The important constraint is honesty: OpenVPS is a batteries-included research tool and deployable foundation, but not a managed cloud product. That distinction is intentional and should remain visible on the site.
Origin story

Two grant phases, one open source system.

The first phase, Aurora Viewer under NGI Atlantic, proved the OSCP concept on a live testbed. The second phase, OSCAR4US under NGI Search, turned that proof of concept into the current OpenVPS stack built at Nokia Bell Labs for OARC Europe.

This history matters because it explains why the project is open, why the implementation is real, and why the site credits both the research lineage and the production engineering team.

2022

Aurora Viewer, NGI Atlantic

George Mason University, OARC Europe, 3DInteractive, and COSMOS Lab demonstrated the first open-source VPS reference deployment on a 5G testbed.

2024-2025

OSCAR4US, NGI Search

Nokia Bell Labs built MapBuilder, MapAligner, MapLocalizer, and the GeoPoseProtocol compliance layer with OARC Europe administration.

Ecosystem

OpenVPS fits into the open spatial web stack.

Just as the web needed standards, protocols, services, discovery, and browsers, the spatial web needs GeoPose, GeoPoseProtocol, OpenVPS, OSCP discovery, and a spatial browser. OpenVPS is the localization backbone in that stack.

Standard

OGC GeoPose defines the position and orientation format that OpenVPS returns.

Protocol

OSCP GeoPoseProtocol defines the request and response flow that compliant VPS endpoints implement.

Service layer

OpenVPS is the reference open-source implementation that proves the protocol can be deployed openly and self-hosted.
